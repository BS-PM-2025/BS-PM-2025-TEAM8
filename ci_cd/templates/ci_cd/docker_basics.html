🐳 Docker Basics – Learn Containerization

What is Docker?

Docker is a tool that helps you package your application and everything it needs (like Python, dependencies, and files) into a small unit called a container. A container runs the same way on any computer, so you never have to worry about "it works on my machine but not yours" problems again.

Why should you care?

  • ✅ Run your project on any computer without setup issues
  • ✅ Simulate real-world DevOps workflows
  • ✅ Automate testing and deployment inside a controlled environment

What does Docker do to your repository?

When you include a Dockerfile in your repository, Docker can:

  1. Read the Dockerfile to know how to install dependencies
  2. Build an image – a snapshot of your app environment
  3. Run a container – a working instance of your app
This means your code can run in an isolated, consistent way anywhere — even in production.

📦 Common Docker Commands to Try:


# Pull a base Python image
docker pull python

# Run a basic Python shell in a container
docker run -it python

# Build a Docker image from your repo
docker build -t my-ci-app .

# Run your app in a container
docker run my-ci-app

# List all containers
docker ps -a

# Stop and remove containers
docker stop 
docker rm

Try these commands locally if you have Docker installed. It's a great way to understand containerization by doing.
