📘 Introduction to CI/CD

Understand the core concepts of Continuous Integration & Continuous Deployment.

🔧 What is CI?

Continuous Integration (CI) is the practice of automating the integration of code changes. Developers frequently push code to a shared repository, where it's automatically tested and built.

🚀 What is CD?

Continuous Deployment (CD) ensures that code changes are automatically delivered to production after passing tests. This allows faster and more reliable delivery of features.

💡 Real-Life Example

Think of a developer pushing code → it triggers automated testing in Jenkins → if successful, Docker builds a container → then Kubernetes deploys it live. All hands-free!
